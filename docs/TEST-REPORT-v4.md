## 测试中发现并修复的 bug

1. **权限"允许"= 取消**（严重）：kimi 选项为 `approve_once/approve_always/reject`，语义在 `kind` 字段；Tran 按 `allow_*` 匹配 optionId 恒落空 → 全部回落 cancelled。已按 kind 重写映射（`permissionOptionId`）
2. **Usage 预览卡不可见**（用户实测发现）：状态栏 40px 高 `overflow:hidden` 容器把 absolute 卡片整条裁剪。改为 createPortal + fixed 定位
3. **思考碎块**（上迭代已修，本轮回归验证通过）：一 turn 一思考块

## 已知限制（非 bug）

- 旧会话消息预览为空（ACP 无逐条读历史接口）
- 上下文占比依赖 `/usage` 宿主直返行为（kimi 若改返回格式，第三环置灰）
- token 过期需终端 `kimi login` 后重启

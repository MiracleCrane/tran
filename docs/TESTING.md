# Tran 功能测试用例

> 执行方式：CDP 驱动（DevTools 协议连 9223 端口）+ 主进程日志 + 截图核验。
> 环境：`npm run dev` 开发实例，本机 kimi CLI 已登录。

## 1. 会话生命周期

| 编号 | 用例 | 步骤 | 预期 |
|---|---|---|---|
| L1 | 新建对话发消息 | 输入文本回车 | 消息出现在对话区，状态变"处理中"，出现停止按钮 |
| L2 | 流式接收 | 观察回复过程 | 文本逐字流出，末尾有呼吸光标，结束后光标消失 |
| L3 | 中断 | 流式中点停止 | 生成中断，状态复位，可继续发送 |
| L4 | 连续多轮 | 同会话发 3 轮 | 上下文连贯（能引用前轮内容） |
| L5 | 新建第二会话 | 点"新建对话"再发消息 | 两个会话独立，侧栏均可见 |
| L6 | 自动保存 | 聊完一轮后看侧栏 | 无需重启应用，新会话出现在"最近会话" |
| L7 | 历史恢复 | 点侧栏旧会话发消息 | agent 上下文连续（能引用旧内容）；消息预览为空是已知限制 |

## 2. 流式渲染

| 编号 | 用例 | 预期 |
|---|---|---|
| R1 | 思考块 | 一个 turn 只渲染一个思考块（不得一词一块），内容完整 |
| R2 | 思考块折叠 | 默认收起显示摘要，点击展开；展开态固定 max-height，流式自动滚底 |
| R3 | 工具卡折叠 | 单个工具调用默认收起显示摘要行，点击展开详情 |
| R4 | 工具分组 | ≥2 个连续工具调用聚成"N 个工具调用"分组块，含失败时有标识 |
| R5 | Markdown/代码 | 代码块高亮、列表/加粗正常渲染 |

## 3. 工具与文件系统

| 编号 | 用例 | 预期 |
|---|---|---|
| T1 | 读文件 | 让 agent 读存在的文件 → 工具卡成功态，内容正确 |
| T2 | 读不存在文件 | 工具卡失败态，错误信息可见，agent 能恢复继续 |
| T3 | 写文件 | 让 agent 创建文件 → fs/write_text_file 反向请求落盘，磁盘文件真实存在 |
| T4 | 权限审批（default 模式） | 触发需审批的工具 → 弹窗出现，允许/拒绝均生效 |
| T5 | 子代理 | 让 agent 派子代理 → 专属"子代理"卡片（description 标题 + 类型徽章 + 可折叠指令），状态条"子 Agent (1)"出现 |

## 4. 模式与配置

| 编号 | 用例 | 预期 |
|---|---|---|
| M1 | 计划 toggle | 开 → 权限下拉同步"计划模式"；关 → 回 default |
| M2 | 计划模式行为 | plan 下 agent 只读不写（不执行写文件） |
| M3 | 待办卡片 | 触发 plan 事件 → "待办 N 项"卡片出现并实时更新进度 |
| M4 | 模型切换 | 会话中切模型（K3/K2.7）→ 会话不重启、上下文不断 |
| M5 | yolo 模式 | 选 YOLO → 工具调用不再弹审批 |

## 5. 斜杠命令与 Usage

| 编号 | 用例 | 预期 |
|---|---|---|
| S1 | 命令菜单 | 输入 `/` → Kimi 命令组（usage/compact/status 等 15 个）在模板组上方 |
| S2 | /usage | 打开非模态预览卡，不阻断对话区 |
| S3 | Usage 圆环 | 状态栏圆环显示 5h/周额度百分比；悬停出预览卡（无 backdrop 虚化）；数据含重置倒计时 |
| S4 | /status | 返回会话状态文本（模型/模式/上下文） |

## 6. 历史与会话列表

| 编号 | 用例 | 预期 |
|---|---|---|
| H1 | 列表显示 | 侧栏列出 kimi CLI 历史会话（cwd 匹配当前项目） |
| H2 | 标题 | 会话标题有意义（首条消息兜底），不再全是 "New Session" |

## 7. 性能与稳定

| 编号 | 用例 | 预期 |
|---|---|---|
| P1 | 帧率 | 前台空闲 fps avg ≥ 57（CDP rAF 采样 4s） |
| P2 | 长任务 | 3 分钟以上任务流式稳定、可中断、状态条正确 |
| P3 | 托盘 | 关窗最小化到托盘（如设置开启），后台任务完成有原生通知 |

## 已知限制（非 bug）

- ACP 无逐条读历史接口：点开旧会话消息预览为空（agent 侧上下文完好）
- kimi 0.26 ACP 不下发 usage_update：上下文容量环无数据
- token 过期需在终端 `kimi login` 后重启 Tran
